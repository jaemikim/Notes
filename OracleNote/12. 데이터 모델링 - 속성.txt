1. 속성(Attribute)
  1) 인스턴스에서 관리하고자 하는 것
    - 비즈니스에서 필요로 함
  2) 의미상 더 이상 분리되지 않는 최소의 데이터 단위
  3) 엔터티, 인스턴스, 속성, 속성값의 관계
    - 1개의 엔터티는 여러 개의 인스턴스를 가질 수 있고
      하나의 인스턴스는 여러 개의 속성을 가짐
    - 또한 하나의 속성은 단 하나의 속성값을 가지게 됨

2. 속성의 표기법
  1) "#"을 붙여 식별자 임을 표시    // 실질식별자를 선택하면 #이 붙으면 기본키가 된다. 
  2) "*"을 붙여 필수값 임을 표시
  3) "○"을 붙여 선택값 임을 표시 

3. 속성의 특징
  1) 엔터티와 마찬가지로 반드시 비즈니스에서 필요로하고 
     IT 시스템에서 저장 및 관리하고자 하는 정보여야 함
  2) 정규화 이론에 따라 속성이 있는 엔터티의 주식별자에 함수적 종속성을 가져야 함
    - 예) 지하철역 엔터티의 식별자 지하철역번호가 선명과 역명을 결정함   (함수적 종속성이 있다고 볼 수 있음)
  3) 하나의 속성에는 한 개의 값만을 가짐

4. 속성의 분류
  1) 특성에 따른 분류
    - 기본 속성 (Basic Attribute)
      ㆍ비즈니스 분석을 통해 도출된 속성
      ㆍ예) 상가 엔터티의 상호명 속성은 기본 속성임
    - 설계 속성 (Designed Attribute)  
      ㆍ데이터 모델 설계를 하면서 도출하는 속성
      ㆍ예) 상가 엔터티의 표준산업분류코드 속성은 설계 속성이라고 할수 있음
    - 파생 속성 (Derived Attribute)
      ㆍ다른 속성에 의해서 계산이나 변형이 되어 생성되는 속성
      ㆍ예) 상가 엔터티의 주소정보를 기반으로 위도, 경도 속성의 값을 구한다고 하면, 
            위도, 경도 속성은 파생속성이라고 할수 있음 

  2) 엔터티 구성 방식에 따른 분류
